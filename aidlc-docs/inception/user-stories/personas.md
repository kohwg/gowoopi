# User Personas - 테이블오더 서비스

## Persona 1: 김민수 (Customer - 테이블 고객)

### 기본 정보
- **이름**: 김민수
- **나이**: 28세
- **직업**: 회사원
- **역할**: 테이블 고객 (Customer)

### 주요 목표
- 빠르고 편리하게 음식 주문
- 대기 시간 없이 즉시 주문
- 주문 내역 확인 및 추가 주문

### 특성
- **기술 수준**: 중급 (스마트폰 사용에 익숙)
- **사용 빈도**: 주 2-3회 외식
- **선호 방식**: 직관적이고 간단한 UI
- **제약사항**: 
  - 복잡한 로그인 절차 싫어함
  - 빠른 주문 선호
  - 터치 기반 인터페이스 선호

### 동기
- **편의성**: 직원을 부르지 않고 자유롭게 주문하고 싶음
- **속도**: 메뉴 선택부터 주문까지 빠르게 완료하고 싶음
- **투명성**: 내가 주문한 내역을 명확히 확인하고 싶음
- **자율성**: 원하는 시간에 추가 주문하고 싶음

### 사용 시나리오

#### 시나리오 1: 첫 방문 및 주문
1. 테이블에 앉아 태블릿을 확인
2. 자동으로 로그인된 메뉴 화면 확인
3. 카테고리를 탐색하며 메뉴 선택
4. 장바구니에 메뉴 추가 및 수량 조절
5. 주문 확정 후 주문 번호 확인
6. 메뉴 화면으로 돌아가 추가 주문 대기

#### 시나리오 2: 추가 주문
1. 식사 중 추가 메뉴 필요
2. 태블릿에서 메뉴 화면 확인
3. 원하는 메뉴를 장바구니에 추가
4. 주문 확정
5. 주문 내역에서 이전 주문과 새 주문 모두 확인

#### 시나리오 3: 주문 내역 확인
1. 주문한 음식이 언제 나올지 궁금
2. 주문 내역 화면으로 이동
3. 각 주문의 상태 확인 (대기중/준비중/완료)
4. 총 주문 금액 확인

### 사용 패턴
- **접속 시간**: 식사 시간 (점심 11:30-13:30, 저녁 18:00-21:00)
- **세션 길이**: 평균 30-60분
- **주문 빈도**: 세션당 1-3회 주문
- **주요 액션**: 메뉴 탐색 → 장바구니 추가 → 주문 확정 → 주문 내역 확인

### 기대 경험
- 태블릿을 켜면 즉시 메뉴가 보임 (로그인 불필요)
- 메뉴 이미지와 설명이 명확함
- 터치 버튼이 크고 누르기 쉬움
- 주문 후 즉시 피드백 (주문 번호 표시)
- 주문 상태를 실시간으로 확인 가능

### 불만 요소
- 복잡한 로그인 절차
- 느린 응답 속도
- 작은 버튼으로 터치 실수
- 주문 후 피드백 없음
- 이전 주문 내역을 볼 수 없음

## Persona 2: 박지영 (Admin - 매장 관리자)

### 기본 정보
- **이름**: 박지영
- **나이**: 35세
- **직업**: 레스토랑 매니저
- **역할**: 매장 관리자 (Admin)

### 주요 목표
- 실시간으로 들어오는 주문 모니터링
- 주문 상태 관리 및 업데이트
- 테이블별 주문 현황 파악
- 메뉴 관리 및 업데이트

### 특성
- **기술 수준**: 중급 (업무용 소프트웨어 사용 경험)
- **사용 빈도**: 매일 8-10시간 (영업 시간 내내)
- **선호 방식**: 한눈에 파악 가능한 대시보드
- **제약사항**:
  - 바쁜 영업 시간에 빠른 대응 필요
  - 여러 테이블을 동시에 관리
  - 실수 없이 정확한 주문 처리 필요

### 동기
- **효율성**: 주문을 빠르게 확인하고 처리하고 싶음
- **정확성**: 주문 누락이나 실수를 방지하고 싶음
- **가시성**: 모든 테이블의 상황을 한눈에 보고 싶음
- **통제력**: 주문 상태를 직접 관리하고 싶음

### 사용 시나리오

#### 시나리오 1: 영업 시작 및 로그인
1. 매장 오픈 전 관리자 시스템 접속
2. 매장 식별자, 사용자명, 비밀번호 입력
3. 로그인 후 대시보드 확인
4. 영업 시간 내내 로그인 상태 유지 (자동 갱신)

#### 시나리오 2: 실시간 주문 모니터링
1. 대시보드에서 테이블별 카드 확인
2. 신규 주문이 들어오면 시각적 강조 표시
3. 주문 카드 클릭하여 상세 내역 확인
4. 주문 상태를 "대기중" → "준비중" → "완료"로 변경
5. 다음 주문 처리

#### 시나리오 3: 테이블 관리
1. 고객이 식사를 마치고 떠남
2. 해당 테이블의 "이용 완료" 버튼 클릭
3. 확인 팝업에서 확인
4. 테이블 주문 내역이 과거 이력으로 이동
5. 테이블이 새 고객을 위해 리셋됨

#### 시나리오 4: 주문 삭제 (직권 수정)
1. 고객이 주문 취소 요청
2. 해당 주문의 삭제 버튼 클릭
3. 확인 팝업에서 확인
4. 주문이 삭제되고 테이블 총액 재계산

#### 시나리오 5: 메뉴 관리
1. 메뉴 관리 화면으로 이동
2. 신규 메뉴 등록 (메뉴명, 가격, 설명, 카테고리, 이미지 URL)
3. 기존 메뉴 수정 (가격 변경, 설명 업데이트)
4. 품절 메뉴 삭제
5. 메뉴 노출 순서 조정

#### 시나리오 6: 과거 주문 내역 조회
1. 특정 테이블의 과거 내역 버튼 클릭
2. 날짜 필터 선택
3. 과거 주문 목록 확인 (시간 역순)
4. 각 주문의 상세 정보 확인
5. 닫기 버튼으로 대시보드 복귀

### 사용 패턴
- **접속 시간**: 영업 시간 전체 (10:00-22:00)
- **세션 길이**: 8-10시간 (영업 시간 내내)
- **주요 액션**: 
  - 주문 모니터링 (지속적)
  - 주문 상태 변경 (주문당 1-3회)
  - 테이블 이용 완료 (테이블당 1회)
  - 메뉴 관리 (주 1-2회)
- **피크 타임**: 점심(12:00-13:00), 저녁(19:00-20:00)

### 기대 경험
- 로그인 후 자동 갱신으로 계속 로그인 유지
- 신규 주문이 즉시 표시됨 (2초 이내)
- 테이블별로 주문이 그룹화되어 보임
- 주문 상태 변경이 즉시 반영됨
- 한눈에 모든 테이블 상황 파악 가능
- 메뉴 관리가 직관적이고 간단함

### 불만 요소
- 주문 표시 지연
- 복잡한 주문 상태 변경 절차
- 테이블별 주문을 찾기 어려움
- 로그인이 자주 만료됨
- 메뉴 관리가 복잡함
- 과거 내역을 찾기 어려움

## Persona Mapping to User Stories

### Customer (김민수) 관련 스토리
- 자동 로그인 및 세션 관리
- 메뉴 조회 및 탐색
- 장바구니 관리
- 주문 생성
- 주문 내역 조회

### Admin (박지영) 관련 스토리
- 매장 인증
- 실시간 주문 모니터링
- 주문 상태 관리
- 테이블 관리 (초기 설정, 이용 완료, 주문 삭제)
- 메뉴 관리
- 과거 주문 내역 조회
